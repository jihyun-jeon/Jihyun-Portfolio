개발 내용

1. 반응형 사이트 구현
PC, Tablet, Mobile 환경에 대응할 수 있는 반응형 사이트로 구현하였습니다.
또한 PC의 경우 모니터 화면이 늘어나도 컨텐츠가 중앙에 유지되게끔 구현하였습니다.

처음엔 반응형 사이트를 염두하지 않고 시작한터라, 나중에 반응형 구현시 브라우저 사이즈가 줄어들면컨텐츠의 가로 길이가 px로 고정되있어서 뷰포트 밖을 벗어나는 문제가 있었습니다.

이를 해결하기 위해 컨텐츠의 가로 사이즈를 브라우저 기준상대 사이즈인 %로수정하여 해결하였습니다.
2. IntersectionObserver을 활용한 성능 최적화
페이지 스크롤시 특정 위치에 따라 Nav메뉴에 underline 처리를 하기 위해
IntersectionObserver을 활용하여 관찰 요소를 지정하는 방식으로 구현하였습니다.

이를 통해 document에 직접 scroll 이벤트를 바인딩 하던 방식에 비해 더 성능을 최적화 할 수 있었습니다.

scroll 이벤트가 불필요하게 발생하는 현상을 막을 수 있고,
getBoundingClientRect를 호출하지 않아리플로우 현상을 막을 수  있어
사이트의 성능을 좀 더 개선시킬 수 있었습니다.
3. lazyload를 활용한 로딩속도 향상
Carousel를 넘길때 다음 슬라이드의 동영상을 로드 하는데 시간이 오래 걸려 빈 화면인 상태에서 기다려야 했습니다.

슬라이드 이동시동영상 로드 속도를 개선하기 위해 react-slick 기능 중 lazyload 기능을 활용하였습니다.

lazyload 속성 중 anticipated 값을 적용하여 슬라이드를 넘기기 전에 미리 이후 슬라이드 동영상을 로드해주어 로딩 속도를 개선할 수 있었습니다.

성장 포인트

server에서 pre-render한 tree와 브라우저에서 초기 render한 tree가 달라서

React Hydration Error 오류 가 났습니다.


ul태그 없이 li태그를 써서 발생한 문제였고 html 요소 순서를 올바르게 수정하여 해결하였습니다.


이 에러를 해결하면서 hydration을 통해 server에서 내려준 HTML dom을

client에서 매칭하며 렌더링 하고 이후 이벤트 핸들러를 바인딩 하게 되는 원리를 알게되었습니다.
