개발 내용

1. React Hook Form 라이브러리를 활용한 로그인 구현
로그인 폼 유효성 검사시 좀 더 직관적이고 간결한 코드를 쓸 수 있어서 React Hook Form
라이브러리를 사용하였습니다.

📍 커스텀 부분

- 이메일과 비밀번호 필수 입력, 입력 길이 검사, 정규식을 활용한 비밀번호 유효성 검사

- 유효성 검사 오류시 각 상황에 맞는 에러 메세지 출력

2. 페이징 히스토리 연동 처리
선택된 필터 항목 값으로 쿼리 주소를 바꾸고, 브라우저의 주소가 이동되면 해당 쿼리 값을 URLSearchParams로 읽어서 해당 페이지에 맞는 리스트를 추려서 보여줄 수 있도록 구현하였습니다.
3. Redux Toolkit을 사용한 상태관리
사용자 리스트 데이터를 store로 관리하여 고객명 수정, 사용자 추가, 사용자 삭제 시 자동으로 리렌더링 되도록 처리하였습니다.

성장 포인트

1. 상황

이 프로젝트는 원티드 측에서 구축해준 json server를 통해 서버를 실행했습니다.

그런데 구축된 데이터가 깔끔하지 않아 데이터를 조작해서 써야하는 상황이 빈번했으나 팀원 모두는 서버가 그렇게 구축돼있으니 어쩔 수 없는 것으로 알고 진행하였습니다.


2. 과정

그러나 저는 이 상황을 해결할 방법이 있을지 모색해 보았습니다.

구글링을 통해 json server에 대해 알아보았고 검색 결과 api호출 주소에 expand를 넣어서

여러 데이터를 부모 자식 관계로 받을 수 있다는 것을 알아내어 이를 팀원에게도 공유하였습니다.


3. 결과

다른 팀원들도 이러한 방식을 사용하여 좀 더 코드를 깔끔하게 작성할 수 있게 되었습니다.
저는 이렇게 불편한 상황을 문제로 바라보고 그 문제를 해결하는 개발자가 되고자 합니다.
